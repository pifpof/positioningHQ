# PositioningHQ Vault Structure

This document defines the canonical folder layout for a PositioningHQ vault and the purpose of each folder. Claude reads this file during `setup-positioning-vault` to know exactly what to create and what each folder is for.

---

## Full Layout

```
VAULT_ROOT/
├── memory/
│   ├── company-research/
│   └── positioning-playbook/
├── inputs/
│   ├── sales collateral/
│   ├── competitive-intel/
│   ├── roadmap/
│   ├── help-docs/
│   ├── customer references/
│   └── company strategy/
├── outputs/
│   ├── battlecards/
│   ├── call-plans/
│   └── launch/
└── vault.config.md
```

---

## Folder Purposes

### `memory/`

Persistent context that skills read automatically. Think of this as the vault's brain.

| Folder | Written by | Read by |
|---|---|---|
| `memory/company-research/` | `/company-research` | `/positioning-playbook`, `/battlecard` |
| `memory/positioning-playbook/` | `/positioning-playbook` | `/battlecard`, `/call-plan`, `/product-launch` |

---

### `inputs/`

Where you drop reference materials before running skills. Skills automatically scan the relevant `inputs/` subfolders before generating output — the more you put here, the richer the output.

| Subfolder | What to put here |
|---|---|
| `sales collateral/` | Sales decks, one-pagers, product sheets |
| `competitive-intel/` | Analyst reports, competitor tear-downs, win/loss data |
| `roadmap/` | Product roadmap docs, feature lists, release notes |
| `help-docs/` | Knowledgebase articles, product FAQs, support docs |
| `customer references/` | Case studies, testimonials, customer quotes |
| `company strategy/` | Board decks, investor memos, strategy documents |

Add files here anytime — skills will pick them up on the next run.

---

### `outputs/`

Deliverable files generated by skills. These are the files handed to sales, marketing, or shared with stakeholders.

| Folder | Written by | Contains |
|---|---|---|
| `outputs/battlecards/` | `/battlecard` | `battlecard-[competitor].md` per competitor |
| `outputs/call-plans/` | `/call-plan` | `call-plan-[prospect]-[stage]-[date].docx` |
| `outputs/launch/` | `/product-launch` | 6 launch files + positioning update recommendation |

---

### `vault.config.md`

A lightweight tracker file at the vault root. Stores:
- Company name, website, and industry
- Vault creation date and root path
- Checkbox status for each skill that has been run

Skills update this file automatically when they complete. You can also edit it manually.

---

## Naming Conventions

- Company research files: `[CompanyName].md` (e.g. `Stripe.md`)
- Positioning playbook files: `[CompanyName].md`
- Battlecard files: `battlecard-[competitor-name].md` (e.g. `battlecard-hubspot.md`)
- Call plan files: `call-plan-[prospect-company]-[stage]-[YYYY-MM-DD].docx`
- Launch files: prefixed with `launch-` (e.g. `launch-messaging-narrative.md`)
