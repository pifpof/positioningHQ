Setup Positioning
Vault

The onboarding skill. In a single guided session, it creates your vault folder structure, collects company details, and automatically runs /company-research and /positioning-playbook — so every other skill has context from day one.

$/setup-positioning-vault

7-Step Guided Workflow

1

Choose your vault root

Provide any absolute or relative path — e.g. /Users/yourname/vaults/acme or ./acme-vault. Defaults to ./positioning-vault if not specified. This is the only required input before setup begins.

2

Create folder structure

Creates all required folders inside your vault root. See the vault layout below. Skips any folders that already exist.

3

Write vault.config.md

Creates a lightweight tracker file at the vault root. Records company info, creation date, and a checkbox for each skill that has been run. Auto-updates as skills complete.

4

Collect company information

Asks for 5 required fields: company name, website, industry, one-line product description, and 2–3 main competitors. All five are required before research begins.

5

Run /company-research

Executes the full 9-step company research workflow automatically. Stores output in memory/company-research/[CompanyName].md. Summarises key findings and asks if you want to provide additional materials.

6

Run /positioning-playbook

Generates the full 7-section positioning playbook using the research output. Stores in memory/positioning-playbook/[CompanyName].md.

7

Vault ready confirmation

Prints a summary of what was created and what skills you can run next — all of which will auto-load context from your vault.

Vault Folder Layout

The skill creates this structure inside whatever path you specify as the vault root.

VAULT_ROOT/
├── memory/
│ ├── company-research/← output of /company-research
│ └── positioning-playbook/← output of /positioning-playbook
├── inputs/← drop reference materials here anytime
│ ├── sales collateral/
│ ├── competitive-intel/
│ ├── roadmap/
│ ├── help-docs/
│ ├── customer references/
│ └── company strategy/
├── outputs/
│ ├── battlecards/← output of /battlecard
│ ├── call-plans/← output of /call-plan
│ └── launch/← output of /product-launch
└── vault.config.md← status tracker, updated automatically

vault.config.md — Status File

Created automatically and updated each time a skill completes. Check it anytime to see your vault status.

# PositioningHQ Vault

**Created:** 2026-03-12
**Vault Root:** /Users/yourname/vaults/acme

## Company
- **Name:** Acme Corp
- **Website:** acmecorp.com
- **Industry:** B2B SaaS, revenue intelligence

## Status
- [x] Company Research
- [x] Positioning Playbook
- [ ] Battlecards
- [ ] Call Plans
Reinitialise detection: If a vault already exists at the path you provide (detected via vault.config.md), the skill will ask whether to reinitialise or update — it won't silently overwrite your existing memory files.

What Gets Populated

Company Research
